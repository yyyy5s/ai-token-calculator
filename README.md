# ModelCost 🧮

一个在浏览器里快速使用的 AI 模型费用计算器。输入模型、Token 用量、缓存命中率和调用次数后，ModelCost 会估算单次、每日、每月成本，并支持用 USD / CNY 查看结果，可添加自定义货币。

它适合在选型、报价、预算预估、模型比价、账单复盘时快速回答一个问题：**这个模型大概要花多少钱？**

## ✨ 功能亮点

- 🧾 **单次 / 每日 / 每月成本估算**：输入一次请求的大致用量，即可估算长期费用。
- 🔁 **从总量反推单次成本**：已有日志或账单时，可以填入总输入、总输出和调用次数，自动折算平均成本。
- 💰 **USD / CNY 双币种显示**：自动获取美元兑人民币汇率，也可以手动调整，添加新币种。
- 🧠 **支持缓存价格**：可以填缓存命中率，估算使用上下文缓存后的费用变化。
- 📊 **模型横向对比**：同一组参数下，最多对比 5 个模型的成本差异。
- 🧩 **自定义模型与币种**：内置列表不够时，可以添加自己的模型价格和计费币种。
- 📦 **导入 / 导出配置**：自定义模型和币种可以导出为 JSON，方便迁移或备份。
- 🌗 **浅色 / 深色主题**：支持主题切换。
- 🌐 **多语言界面**：可在页面顶部切换语言。
- 📱 **可安装为 PWA**：在支持的浏览器中可安装到桌面或移动设备。

## 适合谁使用

ModelCost 面向需要粗略或半精确估算 AI 调用成本的人，包括：

- 正在比较不同 AI 模型费用的开发者、产品经理或创业团队；
- 需要给客户、项目或内部预算做报价的人；
- 想知道缓存、长上下文、输出长度会怎样影响费用的人；
- 已经有调用日志，想把总 Token 用量换算成平均单次成本的人。

## 快速开始 🚀

这是一个纯前端项目，不需要后端服务。

### 方法一：直接打开

```bash
git clone https://github.com/yyyy5s/model-cost.git
cd model-cost
```

然后用浏览器打开 `index.html`。

### 方法二：本地静态服务

如果浏览器限制本地文件读取，可以启动一个简单的静态服务：

```bash
python -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```

## 怎么使用

### 1. 选择模型

从左侧「模型库」选择一个内置模型。也可以点击「添加」，创建自己的模型价格。

### 2. 选择输入方式

ModelCost 提供两种方式：

| 模式 | 适合场景 | 需要填写 |
| --- | --- | --- |
| 单次算总量 | 你知道一次请求大概有多少输入和输出 | 单次输入 Token、单次输出 Token、每日调用次数 |
| 总量算单次 | 你手上有一段时间的日志或账单 | 总输入命中 Token、总输入未命中 Token、总输出 Token、调用次数 |

### 3. 填 Token 用量

可以把 Token 理解为 AI 服务的计费单位。通常：

- **输入 Token**：你发给模型的内容，包括提示词、历史对话、系统提示词、上下文资料等。
- **输出 Token**：模型生成的回复内容。
- **缓存命中率**：有多少输入内容被缓存复用。命中率越高，支持缓存计费的模型通常越便宜。

不确定 Token 数时，可以先用粗略估计，等有日志后再用「总量算单次」修正。

### 4. 查看结果

页面会显示：

- 单次成本；
- 每日成本；
- 每月成本；
- 输入 / 输出成本占比；
- 同一参数下多个模型的横向对比。

## 自定义模型价格

点击左侧「添加」可以录入自己的模型。所有价格都按「每 100 万 tokens」填写。

| 计费方式 | 什么时候用 | 填写方式 |
| --- | --- | --- |
| 固定单价 | 输入、输出价格不随长度变化 | 填输入价格、输出价格、可选的缓存命中价格 |
| 按输入量分档 | 单次输入越长，整次请求使用另一档价格 | 每一档填写输入上限，以及该档输入 / 输出 / 缓存价格 |
| 按输出量分档 | 单次输出越长，整次请求使用另一档价格 | 每一档填写输出上限，以及该档输入 / 输出 / 缓存价格 |

填写提示：

- 价格单位是「当前币种 / 100 万 tokens」，不是「每 1K tokens」。
- 如果官方价格写的是 `$0.15 / 1M tokens`，就填 `0.15`。
- 如果官方价格写的是 `$0.00015 / 1K tokens`，换算成每百万就是 `0.15`。
- 如果模型没有公开缓存命中价格，可以留空。
- 最后一档没有上限时，阈值留空即可。

## 数据与隐私

- 自定义模型、币种、偏好设置和对比列表保存在浏览器本地。
- 页面会请求公开汇率接口获取 USD → CNY 汇率；如果请求失败，会使用缓存汇率或默认值。
- 本项目不需要账号，也不需要把你的模型配置上传到服务器。

## 多语言支持与翻译贡献 🌐

ModelCost 目前内置中文和英文界面，所有界面文案集中维护在：

```text
data/i18n.js
````

如果你想新增语言或改进现有翻译，可以按下面的方式处理。

### 新增一种语言

1. 打开 `data/i18n.js`。
2. 参考现有的 `zh` 和 `en` 结构，新增一个语言对象，例如：

```js
ja: {
  // 按照 zh / en 的 key 补充对应翻译
}
```

3. 确保新增语言包含和现有语言完全一致的 key。
4. 在语言切换入口中加入新语言选项。
5. 本地打开页面，检查所有页面文案、弹窗、按钮、帮助内容是否显示正常。

### 多语言检查清单

提交翻译前，请检查：

* 页面顶部语言切换是否正常；
* 长文本在移动端不会明显溢出；

如果只是修正错别字、优化表达或补充帮助说明，也欢迎直接提交。

## 维护模型价格

模型价格变化较快。更新内置模型时，建议同步检查：

- 官方价格是否按输入 / 输出分别计费；
- 是否有缓存命中价格；
- 是否有 Batch、夜间、Flex 等折扣；
- 是否存在长上下文分档；
- 价格单位是每 1K、每 1M，还是按请求收费。

目前 ModelCost 更适合估算文本 Token 成本。图片、音频、工具调用、缓存写入、批处理折扣等复杂计费场景，可以通过「复制模型并调整价格」的方式近似表达。
